Arrays and Hashes

November 15th, 2015

These first four weeks has flown by. After the past couple of weeks focused on HTML, CSS and web design we’ve totally switched gears to Ruby. The last time I had touched Ruby was right before my interview for DBC and it all started to come back to me. One of the big concepts that are widely used in Ruby are Array and Hashes.

Array and Hashes are very similar and that they are both ways to store a collection of data. For example, they can store each grade for each student. An Array and Hash would like the following:

Array = [[“Alex”, 100], [“Joey”, 85]]

grades = Hash.new
grades[“Alex”] = 100
grades[“Joey”] = 85

They both are able to store the information, but the greatest difference is retrieving that information. Arrays are an integer-indexed collection of data, so each element is associated with an integer. Whereas, for a Hash, each value is associated with a Key. In this instance, the key is the students name and is assigned a value. So if we wanted to retrieve Joey’s grade we would do the following:

puts array[1][1]

puts grades[“Joey”]

They both will return 85. Now this wasn’t too bad for the array, but just imagine if it was a class full of 30 students. It would be extremely difficult to retrieve a students information, because it would take time to look up which index to use. For the Hash, all that we need to know is the key and in this case would be the student’s name.

So, I think it would be best to use Hashes when we need to create a large collection of data that we will be needing to retrieve often because of the much easier accessibility of values compared to arrays.
